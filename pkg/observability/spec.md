# observability spec

## 1. 目标

本 spec 只定义当前下一步要完成的行为契约：

- 为 payload 引入大小限制；
- 为 payload 生成预览；
- 为 payload 生成哈希；
- 统一 payload 脱敏策略；
- 为后续将摘要信息挂回 Span attributes / events 提供稳定输入。

本 spec 只定义可测试规则，不定义实现方式。

## 2. 适用范围

本 spec 仅适用于 `pkg/observability` 中与 payload 规范化相关的行为。

本 spec 不覆盖以下内容：

- `doJSONRequest` 的业务接入；
- 流式响应聚合；
- exporter 结构调整；
- `summary.md` 或 `timeline.txt` 的展示格式变更；
- 业务层 span 命名策略。

## 3. 输入对象

本步骤处理的对象称为 `payload input`。

`payload input` 允许为：

- `map[string]any`
- `[]any`
- `string`
- `[]byte`
- `json.RawMessage`
- `nil`

本步骤输出的对象称为 `payload summary`。

`payload summary` 必须至少包含以下语义字段：

- 脱敏后的 payload 值；
- payload 原始字节长度；
- 预览内容；
- 哈希值；
- 是否截断。

字段名可以调整，但语义不可缺失。

## 4. 行为契约

### 4.1 大小统计

- 对任意可接受输入，系统必须计算 payload 的原始字节长度。
- 原始字节长度必须基于进入截断逻辑前的 payload 内容计算。
- `payload == nil` 时，原始字节长度必须为 `0` 或实现定义的等价空值，但同一实现内必须保持一致。

### 4.2 统一脱敏

- 在生成预览前，payload 必须先完成脱敏。
- 在生成哈希前，payload 必须先完成脱敏。
- 在生成用于后续落盘或挂属性的摘要前，payload 必须先完成脱敏。
- 命中脱敏规则的字段不得在预览、哈希输入或后续摘要中保留原始敏感值。

### 4.3 预览

- 系统必须为每个 payload 生成预览内容。
- 预览必须来自脱敏后的 payload。
- 预览必须是稳定字符串；同一输入在同一配置下多次处理，预览结果必须一致。
- 当 payload 为空时，预览必须为空字符串或实现定义的等价空值，但同一实现内必须保持一致。
- 当 payload 超过预览上限时，预览必须被截断。
- 当预览被截断时，结果中必须有可测试的截断标记。

### 4.4 哈希

- 系统必须为每个 payload 生成哈希值。
- 哈希输入必须是脱敏后的 payload，而不是原始 payload。
- 同一脱敏后 payload 在同一实现中多次处理，哈希值必须一致。
- 两个不同的脱敏后 payload 不应产生相同哈希值；若出现碰撞，不作为单次测试必现条件，但实现不得故意返回常量哈希。
- 空 payload 也必须有稳定哈希值，或者返回明确的空哈希约定；同一实现内必须保持一致。

### 4.5 截断

- 系统必须支持 payload 大小上限。
- 当 payload 未超过上限时，结果中的 `truncated` 必须为 `false`。
- 当 payload 超过上限时，结果中的 `truncated` 必须为 `true`。
- 当 payload 超过上限时，后续用于展示或挂属性的 payload 内容不得继续保留完整原文。
- 截断行为不得影响原始字节长度统计。

## 5. 成功条件

当输入合法且脱敏器可用时，处理必须成功并满足以下条件：

- 返回非空结果对象；
- 返回结果包含原始字节长度；
- 返回结果包含预览；
- 返回结果包含哈希；
- 返回结果包含截断标记；
- 预览来自脱敏后 payload；
- 哈希基于脱敏后 payload；
- 大小统计基于截断前输入。

## 6. 失败条件

当出现以下任一情况时，处理必须返回错误：

- 输入类型不在允许范围内，且无法转换为受支持类型；
- payload 无法被序列化为可处理内容；
- 脱敏器不可用；
- 脱敏过程失败；
- 预览生成过程失败；
- 哈希生成过程失败。

返回错误时，不得返回看似成功但字段不完整的结果对象。

## 7. 错误语义

- 输入类型错误时，错误必须明确表示“输入类型不支持”或等价语义。
- 序列化失败时，错误必须明确表示“payload 无法序列化”或等价语义。
- 脱敏器不可用时，错误必须明确表示“redactor 不可用”或等价语义。
- 脱敏失败时，错误必须明确表示“脱敏失败”或等价语义。
- 返回错误时，不得静默降级为输出原始敏感内容。

## 8. 边界条件

- `payload == nil` 必须可处理。
- 空字符串必须可处理。
- 空数组必须可处理。
- 空对象必须可处理。
- 原始 JSON 字符串前后带空白时，处理结果必须与去除首尾空白后一致。
- 非 JSON 普通字符串必须可处理，不得因为不是合法 JSON 就直接失败。
- 超大 payload 必须可处理；允许返回截断结果，但不得因为超大直接 panic。

## 9. Forbidden Behaviors

- 禁止在脱敏之前生成预览。
- 禁止在脱敏之前生成哈希。
- 禁止在摘要处理中泄露原始敏感字段值。
- 禁止对超大 payload 无限制保留完整展示内容。
- 禁止返回固定常量哈希来伪造哈希能力。
- 禁止在发生错误时静默返回部分成功结果。
- 禁止通过修改调用方输入对象来完成摘要处理。

## 10. 完成判定

当以下条件全部满足时，本步骤视为完成：

- payload 可得到稳定的原始字节长度；
- payload 可得到脱敏后的稳定预览；
- payload 可得到基于脱敏结果的稳定哈希；
- payload 超过上限时可得到稳定截断结果和截断标记；
- 空值、空字符串、普通字符串、结构化对象和超大 payload 都有明确行为；
- 错误场景有稳定错误语义；
- 不出现本 spec 定义的 Forbidden Behaviors。
